# إعداد قاعدة البيانات للأداء العالي (Scalability Guide)

لضمان تحمل الموقع لآلاف الزوار، يجب الانتقال من قاعدة بيانات SQLite (التي تأتي افتراضياً) إلى MySQL أو PostgreSQL.

## لماذا هذا مهم؟
*   **SQLite:** ملف واحد، يقفل عند كل عملية كتابة (بطيء مع كثرة اللاعبين).
*   **MySQL/PostgreSQL:** أنظمة خادم-عميل، تتحمل آلاف العمليات المتزامنة، وتدعم النسخ الاحتياطي والأداء العالي.

---

## الخطوات على PythonAnywhere

### 1. إنشاء قاعدة بيانات MySQL
1.  اذهب إلى تبويب **Databases** في لوحة تحكم PythonAnywhere.
2.  في قسم **Create a database**، أدخل اسماً للقاعدة (مثلاً: `gangs_db`) واضغط **Create**.
3.  في قسم **MySQL Settings**، قم بتعيين كلمة مرور لقاعدة البيانات (احفظها جيداً).
4.  لاحظ العنوان (Host address) واسم المستخدم واسم القاعدة. عادةً يكون العنوان:
    `Username.mysql.pythonanywhere-services.com`

### 2. ربط الموقع بقاعدة البيانات
اذهب إلى تبويب **Web** في PythonAnywhere، ثم انزل لقسم **Environment variables** (أو عدل ملف `.env` إذا كنت تستخدمه).
أضف المتغير التالي (استبدل القيم بمعلوماتك):

```bash
DATABASE_URL=mysql://Username:DB_PASSWORD@Username.mysql.pythonanywhere-services.com/Username$gangs_db
```
*   `Username`: اسم حسابك في PythonAnywhere.
*   `DB_PASSWORD`: كلمة المرور التي عينتها للقاعدة.
*   `gangs_db`: اسم القاعدة التي أنشأتها (لاحظ أن PythonAnywhere يضيف اسم المستخدم قبل اسم القاعدة، مثل `azad$gangs_db`).

### 3. ترحيل البيانات (Migrations)
بعد ضبط المتغير، افتح **Console** (Bash) ونفذ الأوامر التالية لإنشاء الجداول في القاعدة الجديدة:

```bash
# الانتقال للمجلد
cd ~/GangsOfPalestine

# التأكد من تثبيت المكتبات
pip install mysqlclient

# إنشاء الجداول
flask db upgrade
```

> **ملاحظة:** هذا سينشئ قاعدة بيانات فارغة ونظيفة. إذا كنت تريد نقل بيانات اللاعبين الحاليين من SQLite، ستحتاج لعملية تصدير واستيراد يدوية، ولكن يُفضل البدء بقاعدة نظيفة عند الإطلاق الرسمي لتجنب المشاكل.

### 4. تحسينات الأداء (تم تطبيقها في الكود)
لقد قمنا بتحديث الكود ليشمل:
*   **Faharis (Indexes):** تمت إضافة فهارس ذكية للبحث السريع عن اللاعبين وترتيبهم حسب المستوى والخبرة.
*   **Connection Pooling:** تم ضبط إعدادات الاتصال لتكون مستقرة مع MySQL وتتجنب انقطاع الاتصال (Timeouts).
*   **Caching:** تأكدنا من تفعيل التخزين المؤقت (Cache) للصفحات الثقيلة.

## نصيحة إضافية
إذا زاد عدد اللاعبين عن 10,000 نشط يومياً، فكر في ترقية خطة الاستضافة أو الانتقال لسيرفر خاص (VPS) مثل DigitalOcean أو AWS للحصول على تحكم كامل في موارد قاعدة البيانات.
